BLOCKS

The stars every blockchain are of course, THE BLOCKS! The first of which is known as the genesis block. Every so often a new block is created, or mined (roughly every 10 minutes in the case of Bitcoin), and is added to the blockchain. This block may contain any transactions that were made since the mining of the previous block.

In order to mine a new block, one must solve an extremely difficult puzzle. This puzzle is based on a cryptographic hash algorithm, SHA-256 being one of them. A cryptographic hash acts like a signature for a text, data file, or in this case a block. SHA-256 algorithm generates a random fixed sized (256-bit) hash. A unique set of data will always produce the same unique hash, but even a single change in the data will create a new completely random hash. This and the fact that a hash is a one way function, meaning it cannot be decrypted back, make it suitable for digital signatures.

In each block a certain set of values are run through the hashing algorithm in order to generate a hash. In the example below ( and all subsequent examples ) the block's data, previous blocks hash, and date it was mined are used to create it's hash. In order to add a block to the blockchain it must first be signed, meaning the hash must meet a certain requirement, or difficulty, . This is the Proof-of-Work. In this case a signed hash must start with '000'. In order to achieve this the nonce is also used to create a valid hash. The nonce is an arbitrary number that can only be used once, and is included with the block's data, date mined, and previous hash values in order to find a hash that satisfies the set difficulty. Finding a valid hash, in essence, is the mining process. By increasing the the difficulty of a valid hash the more computational power and time is required to mine a new block.
