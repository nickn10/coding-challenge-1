BLOCKCHAINS

When adding a block to the chain, and to have a valid blockchain, not only must the newly mined block have a valid hash, but it's previous hash value must match the preceding block's hash. This is extremely important as this defines the order of the blocks in the chain. If you add a few blocks to create a chain below, you'll notice that not only does each new block contain a valid hash but it's previous hash value matches the hash of the block before it. You now have a valid blockchain!

Now that you have a valid blockchain with important information in it, what would happen if you attempted to alter that important data in the genesis block to give your self some extra dough. If you guessed you'd mess up your perfectly good blockchain you would be correct! As seen prior, changing the data in a block changes the hash. And since the following block's hash takes into account the previous hash it's hash has changed, and so on for every block that follows in the chain. You could always mine all the blocks in the chain again to make it valid, but that would take an incredible amount of computing power! Even if you where able to fork a blockchain and mine new blocks at a faster pace the the rest of the network, there would still be thousands of copies to compare it against to see that it had been altered.
